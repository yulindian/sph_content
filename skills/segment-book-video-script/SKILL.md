# 图书短视频文案分段

## 核心规则

- 不改变原意，不增删内容，不改写原句。
- 每段目标为50至80个中文字符；标点不计入字数。
- 先保证一个段落内的场景或叙事逻辑闭环，再微调切分点。
- 同一场景的时间、地点、人物和事件应尽量合并。
- 包含“动作+目的+结果”或“铺垫+转折”等完整逻辑的句群应保持在同一段。
- 不在完整词语、书名、人名、数据、固定概念或紧密语义中间切断。
- 不因凑字数制造生硬断点。

## 字数处理

1. 扫描场景变化、时间跳转、人物切换、因果完成和转折位置。
2. 形成初步段落后逐段计数。
3. 对不足50字的段落，优先与前后语义最接近的段落合并并重新平衡。
4. 对超过80字的段落，在最近的自然语义停顿处拆分。
5. 若全文本身不足50字，保留为一段；这是唯一自然的短段例外。
6. 终段出现短尾时，回溯调整前两段，不直接留下可避免的短段。

## 自检

静默确认：

- 所有原始文字均出现且顺序不变。
- 没有重复、遗漏或改写。
- 除全文不足50字外，每段均为50至80字。
- 每段可独立对应一个完整画面或连续镜头。

## 输出

仅在代码块中输出分段结果。每段格式为：

```text
1. 段落内容
2. 段落内容
```

代码块后输出：`总段落数：X段`
