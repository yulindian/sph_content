# 图书爆款文案二创

## 接收输入

要求用户提供：

- 书名
- 作者
- 对标原文

若本次只缺非关键字段，先完成可执行部分并明确标注缺失项；不要自行编造书名、作者、人物、时间、数据或情节。

## 优先级

按以下顺序解决冲突：

1. 用户明确提供的书名、作者必须正确。
2. 原文事实、人物、时间线、精确数据及核心观点不得改变。
3. 黄金开头和转化结尾原样保留。
4. 中段相似度不高于30%，且口播自然。
5. 改写后总字数控制在纠错后原文的±10%。

若原文中的书名或作者与用户提供的信息冲突，列出冲突并统一替换为用户提供的信息；这种必要修正高于“开头和结尾原样保留”。

## 预处理与事实检查

1. 修正明确的错别字、同音字和标点问题。仅在存在问题时输出 `错误 → 修正` 列表。
2. 检查书名、作者在全文中是否一致，有无张冠李戴、前后矛盾或明显事实错误。
3. 对无法由原文或用户信息确认的事实，只标记风险，不擅自修改。
4. 用户要求核实时，优先查询出版社、作者官方资料、图书馆目录等可靠公开来源，并给出来源链接。
5. 后续识别“原样保留区”时，以纠错和元数据修正后的文本为准。

## 识别不可改写区

- 将开篇前3至4句、约黄金3至5秒识别为黄金钩子，逐字保留。
- 将末尾完整转化闭环识别为转化结尾，逐字保留，包括塑品逻辑、购买引导和互动引导。
- 不要把大段普通正文误判为不可改写区。

## 深度重构中段

保留事实、人物、时间线、数据、核心观点、对比框架、五感强度和情绪内核。禁止杜撰原文不存在的对话、数据和情节。

至少组合使用以下三类策略：

- 重组信息顺序和论述推进顺序。
- 转换叙事视角、主被动关系、动作描写、转述或对话形式。
- 在不增加新事实的前提下全面重写句式，拆合长短句，转换陈述、反问和设问。
- 用同等冲击力替换非核心动作、表情或感官细节。
- 仅基于可靠的百科级常识补充背景；无法确认则不补充。

禁止：

- 简单同义词替换。
- 仅在段内调换句子。
- 复制中段任何完整原句。
- 连续三句以上沿用原文结构。
- 使用“在……的世界里”“让我们一起”“值得一提的是”等AI腔。
- 用平淡表达削弱原文的五感和情绪。

保持中文口语化、紧凑、有推进和悬念，确保中老年受众容易听懂。

## 内部自检

交付前静默检查并修正：

- 黄金钩子、塑品逻辑、互动引导、购买引导是否完整。
- 长短句、停顿、语气和朗读节奏是否自然。
- 五感细节、共鸣金句和情绪强度是否得到等强度保留。
- 是否编造事实，或遗漏人物、时间和数据。
- 是否有中段完整原句或连续三句结构相似。
- 书名和作者是否全文一致。
- 字数是否在±10%以内。

不要展示内部检查过程。

## 文皮皮相似度检测

改写后打开 `http://www.wenpipi.com/sim`，以纠错后的对标原文和二创文案进行检测。执行网页操作前，加载可用的 Browser 或 Chrome 控制技能。

- 以 **不高于30%** 为硬性验收标准。
- 若结果高于30%，只继续重构中段，不改动已确认的黄金钩子和转化结尾，然后重新检测。
- 持续迭代至不高于30%。
- 若网站不可访问、要求无法处理的登录或验证码，仍交付文案，并将结果写为“待人工检测”，说明阻塞原因；不得伪造百分比。

相似度百分比是文本检测结果，不代表版权或平台审核结论。

## 输出

依次输出：

1. 仅在发现问题时输出“纠错与一致性检查”。
2. 简要列出至少三种实际采用的重构策略。
3. 输出原文字数、二创字数及浮动百分比。
4. 在 Markdown 代码块中输出完整二创文案。
5. 输出文皮皮检测结果及是否达到30%标准。

## 深度补充模式

用户说“深度补充”或“增加素材”时：

1. 标出可补充的人物背景、事件细节或对比点。
2. 优先使用用户资料；需要公开信息时先说明将核实可靠来源。
3. 只补充可验证事实并附来源；不确定内容跳过，不用“据说”掩盖无依据内容。
4. 重新执行改写、自检和相似度检测。
