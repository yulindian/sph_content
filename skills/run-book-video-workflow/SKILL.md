# 图书视频号全流程

## 必需输入

接收：

- 书名
- 作者
- 对标原文

缺少任一项时只追问缺失项，不开始生成最终结果。

## 加载独立能力

执行前完整读取以下相邻 Skill，并以其最新规则为准：

1. `../remix-book-viral-copy/SKILL.md`
2. `../segment-book-video-script/SKILL.md`
3. `../generate-jimeng-image-prompts/SKILL.md`
4. `../format-book-video-subtitles/SKILL.md`

若独立 Skill 与本文件冲突，以本文件规定的流程顺序和最终表格格式为准；各阶段的内容规则以独立 Skill 为准。

## 执行顺序

不等待阶段确认，直接完成全流程：

1. **校验输入**：纠正明确错别字，检查书名、作者是否全文一致，标记明显事实冲突。用户提供的书名和作者为最高优先级，不静默保留错误书目信息。
2. **爆款二创**：保留黄金开头和完整转化结尾，深度重构中段，保持事实、数据、情绪和核心观点，字数控制在原文±10%。按完整语义组织自然段：一个意思讲完就换行，不把全文连成一段，也不在一句话中间强行断行。
3. **模型原创度自检**：不访问第三方查重网站。执行一次内部检查，排除中段完整原句复用、连续三句结构相似、简单同义词替换和段内调序；发现风险时在本次检查中立即重写。最终记录 `模型原创度自检：通过`，不虚构百分比。
4. **文案分段**：只对最终通过的二创稿分段，通常每段50至80字，保持全文无增删。
5. **生图提示词**：按分段结果逐段生成即梦4:3横屏、8k电影级写实纪录片提示词，数量和序号严格一致。
6. **字幕生成**：用最终二创全文生成无标点字幕；根据句意、口语朗读节奏和配音气口分行，不固定切成9字。普通行以7至9字为最佳且不超过9字；完整书名独占一行，书名超过9字时允许该行超过9字。
7. **视频简介**：基于最终二创文稿生成约100字简介，控制在90至110个汉字。概括核心人物、事件、冲突或阅读价值，语言自然、有吸引力，不添加文稿没有的事实，不堆砌标签，不写购买链接。
8. **一致性终检**：核对书名、作者、事实、段落覆盖、提示词数量、序号、字幕和视频简介，修正后再交付。

## 同步网页看板

完成每份稿件后，查找当前工作区中的 `web/data.js`；若当前目录不是仓库根目录，再查找 `sph_content/web/data.js`。找到后将本次结果追加到 `window.SPH_CONTENT_ROWS`：

```js
{
  id: "稳定且唯一的稿件ID",
  bookTitle: "书名",
  author: "作者",
  videoDescription: "视频简介",
  original: "原文",
  rewritten: "模型原创度自检结果与按完整语义换行的二创文案",
  segmented: "全部分段文案",
  imagePrompts: "全部生图提示词",
  subtitles: "全部字幕"
}
```

- 每份新稿追加一个对象，不覆盖旧对象。
- 修改已有稿件时沿用原 `id` 并更新对应对象，确保浏览器中已保存的备注和制作状态不丢失。
- 正确转义换行、引号和反斜杠，修改后检查 JavaScript 语法。
- 不把备注和制作状态写进 `data.js`；这两项由网页本地存储管理。
- 看板数据文件不存在时跳过网页同步，仍正常输出最终表格。

## 最终输出

只输出一张 Markdown 累计表格。表格固定使用以下八列：

| 书名 | 作者 | 视频简介 | 原文 | 二创 | 分段文案 | 生图提示词 | 字幕 |
|---|---|---|---|---|---|---|---|
| 书名 | 作者 | 约100字简介 | 完整内容 | 完整内容 | 全部段落 | 全部提示词 | 全部字幕 |

格式要求：

- 每份稿件占一行；同一份稿件的全部内容均放在该行对应单元格中。
- 同一对话中用户每新增一份稿件，就保留原有稿件行并在表格末尾追加一行，不覆盖旧行。
- 用户一次提供多份稿件时，每份稿件分别运行完整流程并各占一行。
- 用户明确要求修改已有稿件时，更新对应行，不另增重复行。
- 每类完整内容只占一个单元格，不把分段或字幕拆成多行表格记录。
- 用 `<br>` 表示单元格内换行。
- 将内容中的半角竖线 `|` 转为全角 `｜`，避免破坏表格。
- “视频简介”必须基于该行最终二创稿，控制在90至110个汉字。
- “二创”单元格第一行写 `模型原创度自检：通过`，随后放完整二创文案。
- “二创”必须保留自然语义换行，一个完整意思结束后换行；在 Markdown 表格中用 `<br>` 表示这些换行。
- “分段文案”保留阿拉伯数字序号。
- “生图提示词”保留 `【序号】`。
- “字幕”按句意和配音气口保持一句一行的顺序，以 `<br>` 呈现；普通行不超过9字，完整书名行可超过9字。
- 表格之外不重复粘贴各类内容。
